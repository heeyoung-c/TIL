# 이벤트 핸들링(Event Handling)
`@click === v-on:click`

```vue
<template>
  <button @click="handler">
    Click 1
  </button>
  <button @click="handler">
    Click 2
  </button>
</template>

<script>
export default {
  methods: {
    // 여기서 event 객체는 
    // click과 같은 기타 event의 정보를 가지고 있는 하나의 데이터이다
    // 꼭 event 라는 이름이 아닌, 다른 이름으로 바꿀 수 있다
    handler(event) {
      console.log(event)
      console.log(event.target.textContent)
    }
  }
}
</script>
```

```vue
<template>
  <!-- 
  이때 까지 소괄호를 생략하고 메소드를 호출 한 것은
  vue.js의 문법이었다. 때문에 인수를 넣어주려면 아래와 같이 해야한다 
  -->
  <button @click="handler('hi')">
    Click 1
  </button>
  <button @click="handler('what')">
    Click 2
  </button>
  <!-- 인수를 넣어주지 않으면 event 객체가 들어온다 -->
</template>

<script>
export default {
  methods: {
    handler(msg) {
      console.log(msg)
    }
  }
}
</script>
```

```vue
<template>
  <button @click="handler('hi', $event)">
    Click 1
  </button>
  <button @click="handler('what', $event)">
    Click 2
  </button>
  <!-- 
    두번째 인수로 vue.js에서 제공하는 문법을 사용하면
    event 객체를 동시에 사용할 수 있다
   -->
</template>

<script>
export default {
  methods: {
    handler(msg, e) {
      console.log(msg)
      console.log(e)
    }
  }
}
</script>
```

```vue
<template>
  <!-- 
    만들어진 여러개의 함수를 동시에 실행할 수 있는 방법 
    주의할 점은, 쉼표로 구분하여 여러개를 실행할때는 반드시 ()를 작성해주어야 한다
  -->
  <button @click="handlerA(), handlerB()">
    Click me!
  </button>
</template>

<script>
export default {
  methods: {
    handlerA() {
      console.log('A')
    },
    handlerB() {
      console.log('B')
    } 
  }
}
</script>
```
## 이벤트 수식어

### `prevent`
```vue
<template>
  <a 
    href="https://naver.com"
    target="_blank"
    @click="handler">
    NAVER
  </a>
</template>

<script>
export default {
  methods: {
    handler(event) {
      event.preventDefault()
      // 기본 동작을 방지
      // html에 기본적으로 있는 기능인 NAVER을 눌렀을 때, 이동하는 기능을 방지
      console.log('ABC!')
    }
  }
}
</script>

<!-- prevent 수식어 사용 -->
<template>
  <a 
    href="https://naver.com"
    target="_blank"
    @click.prevent="handler">
    <!-- prevent라는 수식어만 붙이고도 같은 기능을 활용할 수 있다 -->
    NAVER
  </a>
</template>

<script>
export default {
  methods: {
    handler() {
      console.log('ABC!')
    }
  }
}
</script>
```

### `once`

특정 이벤트가 발생했을 때,  
해당 메소드를 단 1번만 실행해주는 수식어

```vue
<template>
  <a 
    href="https://naver.com"
    target="_blank"
    @click.once="handler">
    NAVER
  </a>
</template>

<script>
export default {
  methods: {
    handler() {
      console.log('ABC!')
      // 한번만 출력된다
    }
  }
}
</script>
```

### 수식어 체이닝

```vue
<template>
  <a 
    href="https://naver.com"
    target="_blank"
    @click.prevent.once="handler">
    NAVER
  </a>
</template>

<script>
export default {
  methods: {
    handler() {
      console.log('ABC!')
    }
  }
}
</script>
```

### `stop` - 이벤트 버블링 방지

이벤트 버블링을 방지하는 수식어
> 이벤트 버블링이란, 부모요소와 자식요소가 있을 때,  
  자식요소의 이벤트가 부모요소에게 전파되는 것

```vue
<template>
  <div
    class="parent"
    @click="handlerA">
    <div
      class="child"
      @click="handlerB"></div>
  </div>
</template>

<script>
export default {
  methods: {
    handlerA() {
      console.log('A')
    },
    handlerB(event) {
      event.stopPropagation()
      // 이벤트가 전파되는 것을 방지
      // 즉 이벤트 버블링 방지
      console.log('B')
    }
  }
}
</script>

<style lang="scss" scoped>
  .parent {
    width: 200px;
    height: 100px;
    padding: 10px;
    margin: 10px;
    background-color: royalblue;
    .child {
      width: 100px;
      height: 100px;
      background-color: orange;
    }
  }
</style>

<!-- stop 수식어 사용-->
<template>
  <div
    class="parent"
    @click="handlerA">
    <div
      class="child"
      @click.stop="handlerB"></div>
  </div>  
</template>

<script>
export default {
  methods: {
    handlerA() {
      console.log('A')
    },
    handlerB() {
      console.log('B')
    }
  }
}
</script>

<style lang="scss" scoped>
  .parent {
    width: 200px;
    height: 100px;
    padding: 10px;
    margin: 10px;
    background-color: royalblue;
    .child {
      width: 100px;
      height: 100px;
      background-color: orange;
    }
  }
</style>
```

### `capture` - 이벤트 캡처링 방지 

> 이벤트 캡처링이란  
  이벤트 버블링과는 반대로, 부모요소에서 자식요소로 내려오는 개념

```vue
<!-- B -> A 순서로 출력 -->
<!--
B
A 
-->
<template>
  <div
    class="parent"
    @click="handlerA">
    <div
      class="child"
      @click="handlerB"></div>
  </div>
</template>

<!-- A -> B 순서로 출력 -->
<!--
A
B 
-->
<template>
  <div
    class="parent"
    @click.capture="handlerA">
    <div
      class="child"
      @click="handlerB"></div>
  </div>
</template>

<!-- A만 출력 -->
<!--
A
-->
<template>
  <div
    class="parent"
    @click.capture.stop="handlerA">
    <div
      class="child"
      @click="handlerB"></div>
  </div>
</template>
```

### `self`

`child` 부분을 클릭한다는 것은 곧, `parent` 부분을 클릭한다는 것  
정확하게, 자신이 노출되어 있는 부분을 클릭하였을 때, 작동하게 만들어주는 수식어

```vue
<template>
  <div
    class="parent"
    @click.self="handlerA">
    <!-- 
      self 수식어는 
      target 과 currentTarget이 일치할 때만 
      해당 이벤트 실행
    -->
    <div
      class="child"></div>
  </div>
</template>

<script>
export default {
  methods: {
    handlerA(event) {
      console.log(event.target)
      // target: 실제로 클릭이 된 그 요소를 지칭

      console.log(event.currentTarget)
      // currentTarget: currentTarget이 실행된 해당하는 함수에 연결이 되어있는 그 이벤트에 해당하는 요소 출력

      console.log('A')
    },
    handlerB() {
      console.log('B')
    }
  }
}
</script>

<style lang="scss" scoped>
  .parent {
    width: 200px;
    height: 100px;
    padding: 10px;
    margin: 10px;
    background-color: royalblue;
    .child {
      width: 100px;
      height: 100px;
      background-color: orange;
    }
  }
</style>
```

### `passive`

아래 코드에서 기본적인 로직의 처리와 스크롤을 독립시킨다.  
사용자 경험을 향상시킬 수 있다

```vue
<template>
  <div
    class="parent"
    @wheel.passive="handler">
    <div class="child"></div>
  </div>
</template>

<script>
export default {
  methods: {
    handler(event) {
      for (let i = 0; i < 1000; i += 1) {
        console.log(event)
      }
    }
  }
}
</script>

<style lang="scss" scoped>
  .parent {
    overflow: auto;

    width: 200px;
    height: 100px;
    padding: 10px;
    margin: 10px;
    background-color: royalblue;
    .child {
      width: 100px;
      height: 2000px;
      background-color: orange;
    }
  }
</style>
```

## 키 수식어

```vue
<template>
  <input 
    type="text"
    @keydown="handler" />
</template>
<script>
export default {
  methods: {
    handler(e) {
      if (e.key === 'Enter') {
        console.log('Enter!!')
      }
    }
  }
}
</script>

<!-- 키 수식어 사용 -->
<template>
  <input 
    type="text"
    @keydown.enter="handler" />
</template>
<script>
export default {
  methods: {
    handler() {
      console.log('Enter!!')  
    }
  }
}
</script>
```
중요한 점은, 수식어를 케밥 케이스(-)로 작성을 해야한다
